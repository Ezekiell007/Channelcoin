## How to Report a Vulnerability

If you discover a potential security issue in:

- Smart contracts  
- Website  
- Repository code  
- Integrations  
- APIs  
- Sensitive documentation  

**Do NOT open a public issue.**

Private reporting helps us investigate and fix the issue before it becomes public.

---

## Response Time

We aim to:

- Provide an initial response within **48 hours**  
- Begin investigating immediately  
- Maintain communication with the reporter until the issue is resolved  

---

## After a Report Is Submitted

Once a vulnerability is confirmed:

1. We analyze the impact  
2. We apply the necessary fix  
3. We update the reporter on the status  
4. We release a public security note if needed  

---

## Recognition

We may publicly acknowledge (optional) individuals who responsibly disclose vulnerabilities.  
If you want recognition, please mention it when reporting.

---

## Responsible Disclosure Guidelines

Please avoid:

- Making the issue public before it is fixed  
- Performing tests that may harm users or the network  
- Sharing malicious files  
- Requesting mandatory rewards  

Please follow:

- Provide clear proof-of-concept  
- Include technical details  
- Explain how the issue can be reproduced  
- Keep all communication private until the issue is resolved  

---

## channelGold Security Commitment

We are committed to:

- Maintaining a secure ecosystem  
- Protecting our community  
- Updating contracts and systems when required  
- Following industry best practices  

Thank you for helping strengthen the security of channelGold (CHG).
